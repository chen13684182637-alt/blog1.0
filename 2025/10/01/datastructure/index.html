数据结构概述

  • 所谓数据结构就是把现实世界里的东西抽象到计算机里，通过计算机中数据集合与关系集合抽象出逻辑结构最后做成存储结构，那么现实世界里的具体不同问题就需要通过不同的算法与数据结构共同构成一个程序来解决

顺序表(ArrayList)

  • 线性逻辑结构

    • 一般的线性表：是具有相同的数据类型的n(n>=0)个数据元素的有限序列，n称为表长，当n=0时线性表是空表。
    • L=(a1,a2,a3,a4…an)

  • 操作：增 删 改 查

  • 线性表的存储/物理实现：

    • 顺序存储：基于顺序存储结构实现的线性表 称为顺序表： 用数组来存储线性表
    • int data[maxx];maxx>=n
    • int size;//记录线性表（顺序表）的实际的元素个数

  • 初始化一个空的顺序表：

    • 申请数组所占空间，给size赋初始值

  • 增：

    • 1.在末尾顺序添加：把新数据放在size位置，size++
    • 2.在指定位置i(0<=i<=size)添加一个数据
      • 1）把原来的i~~size-1往后移动一个位置 空出i位置
      • 2）data[i]=k;

  • 查找数据k所在的位置

  • 删除数据k：

    • 1.查找k
    • 2.删除k

  • 优缺点：

    • 优：1.随机访问 首地址+偏移量直接访问某个数据 data[i] 2.存储密度高

    • 缺点：操作繁琐 浪费时间

单链表(LinkList)

  • 链表的结点结构：

    • 数据域：存放数据
    • 指针域：存放下一个数据所在的结点的地址
    • 首元节点：第一个数据所在的结点
    • 头指针：保存链表中第一个节点的地址
    • 头节点：保存首元结点地址的结点

  • 带头节点的单链表：

    • 使得对首元结点的操作和对其他结点的操作 统一起来
    • 使得空链表和非空链表统一起来

  • 不带头节点的单链表

  • 操作：增 删 改 查

    • 1、初始化：创建一个带头结点的空链表L。
      • 声明一个头指针L，创建一个节点空间 指针域赋NULL，头指针L指向该结点空间。
    • 2、增：在链表中添加一个数据k：先新建一个结点s，把数据k放到结点s的数据域中，然后把结点s插入到链表中
      • 头插法：在首元结点前面插入结点s。（1）s的指针域指向原首元结点。（2）头结点的指针域指向新结点s。
      • 尾插法：在尾结点后面插入结点s。（1）先找到尾结点。（2）s的指针域等于尾结点的指针域。（3）尾结点指向s。
      • 在指定位置插入：在数据x之后插入k。（1）找到x所在结点p。（2）s的指针域等于结点p的指针域（3）结点p指向s
    • 3、查找：查找数据x所在的结点，能找到则返回结点的地址，否则返回NULL；
    • 4、删除：在链表中删除数据k
      • 找到k所在的结点p及其上一个结点q
      • q的指针域指向p的下一个结点
      • 释放p结点

  • 优：灵活 插入删除比顺序表快/适用于动态场景

  • 缺：不能随机访问

    • 删：

  • 循环链表：循环单链表 循环双链表

栈(Stack)

  • 栈（Stack）—线性逻辑结构

  • 1.线性表：一对一的线性逻辑结构，可以在任意位置执行任何操作

  • 2.栈：是只允许在一端进行删除或者插入操作的线性表。并且没有查找和修改操作。

    • 术语/概念：
    • 栈顶（Top）：允许执行删除or插入操作的一端
    • 栈底（Bottom）：不允许执行删除or插入操作的一端
    • 空栈：不含任何数据。

  • 性质：先进后出/后进先出

  • 应用场景：

    • 1.函数的调用的实现（递归）
    • 2.业务需求：一些倒着走的逻辑：历史记录，撤销…
    • 3.物理上的堆栈

  • 操作：

    • 初始化一个空栈
    • 入栈：往栈顶放一个数据
    • 出栈：删除栈顶数据

  • 判满

  • 判空

  • 得到栈顶元素（get）

  • 栈的存储结构的实现：

  • 基于顺序存储结构实现：顺序栈——基于数组实现a[maxx]

    • 根据栈顶指针top含义不同 有两种实现方法：

      • 1.栈顶指针top标记真正的栈顶数据：

        • 初始化：声明一个数组，同时把栈顶“指针”top（本质上是下标）赋值为-1.
        • 入栈：先判满，把数据k入栈top++，a[top]=k;
        • 出栈：先判空，top– –；
        • 判满：栈有没有充满整个数组
        • 判空：top= =-1
        • get栈顶元素：先判空，非空时return a[top]

      • 2.栈顶指针top标记真正栈顶数据的下一个位置，即下一个入栈数据的下标：top-1才是真正的栈顶数据

        • 初始化：声明一个数组，同时把栈顶 “指针”top（本质上是下标）赋值为0。

        • 入栈：先判满，把数据k入栈：a[top] = k; top++;

        • 出栈：先判空：top--;

        • 判满：判断栈是否充满整个数组：top == maxx

        • 判空：top == 0

        • get（获取栈顶元素）：先判空，非空时return a[top-1]

  • 基于链式存储结构(带头结点的单链表)实现：链式栈
    • 栈顶指针：头指针，非空时，首元结点是栈顶。尾结点是栈底
    • 初始化：声明一个带头结点的空链表
    • 入栈：头插法
    • 出栈：先判空，删除首元结点
    • 判空：top->next == NULL
    • 上溢：顺序会有，链式不会有
    • 下溢：顺序和链式都会有

队列（Queue）

  • 队列：只允许在一端进行插入操作，另一端进行删除操作的线性表。并且没有查找和修改操作。

  • 队尾：执行插入操作的一端

  • 队首：执行删除操作的一端

  • 空队列：不含任何数据元素

  • 性质：先进先出 / 后进后出

  • 队首 “指针”

  • 队尾 “指针”

  • 入队 出队 判满 判空 初始化

    1. 队列的顺序存储：顺序队列 — 数组 maxx 7

  • data[maxx];

  • 队首指针 int f = 0

  • 队尾指针 int r

  • 单向队列

    • (1) 队尾指针 r 指向真正的队尾元素

      • 初始化: f=0 r=-1
      • 入队: r++ ,data[r]=k;//data[++r]=k
      • 出队：删除队首指针指向的数据。f++
      • 判满：判断数组还能否容纳数据。 r==maxx-1 (可能存在假满) . 假溢出 —– 循环队列
      • 判空: f==r+1

    • (2) 队尾指针 r 指向真正的队尾元素的下一个位置，r-1 是真正的队尾

      • 初始化: f=0 r=0
      • 入队: data[r]=k ,r++. //data[r++]=k
      • 出队：删除队首指针指向的数据。f++
      • 判满：判断数组还能否容纳数据。 r==maxx (可能存在假满) . 假溢出 —– 循环队列
      • 判空: f==r

  • 循环队列：基于循环数组的：通过数学上的取模运算 实现下标的首尾相接

    • 队尾指针 r 指向真正的队尾元素的下一个位置，

    • 初始化: f=0 r=0

    • 入队: data[r]=k,r=(r+1)%maxx,

    • 出队：删除队首指针指向的数据。f=(f+1)%maxx,

    • 判满：

      • (1) 引入变量 sum 记录队列中数据个数

        • r==f && sum==maxx 满

        • r==f && sum==0 空

      • (2) 引入一个标记变量flag==0 出队 flag==1 入队

        • r==f&&flag==1 满。

        • r==f&&flag==0 空

      • (3) 牺牲一个空间，数组中有maxx-1个数据时就满了

        • f== (r+1) %maxx 满

        • r==f 空

  • 2.队列的链式存储：链式队列—–基于带头结点的单链表

    • 首元结点一侧 应该是队首
    • 尾结点一侧 应该是队尾
    • 用头指针做队首指针即可，此时队首指针指向真正队首的上一个结点
    • 队尾指针：指向尾结点的尾指针即可
    • 初始化：搞一个头结点 让队首指针和队尾指针都指向头结点即可
    • 入队：尾插法
    • 出队：删除首元结点 判空：f==r // f->next==NULL

  • 队列的应用场景：一些需要排队的需求

二叉树(Binary Tree)

  • 二叉树：非线性的逻辑结构
    • 特点是每个结点最多只有两颗子树，不存在度大于2度结点。
    • 具有递归性的，是有序树。
  • 特殊的二叉树：
    • 满二叉树：除叶子结点以外，每个结点的度都是2，并且要求叶子结点均在同一层。
      • 高度为h，且第k(1<=k<=h)层均有2^(k-1)个结点的二叉树。
      • 高度为h，且刚好具有(2^h)-1个结点的二叉树
    • 完全二叉树：把满二叉树从上往下 从左往右，从1开始编号。编号之后，从大编号往小编号连续删除若干个结点，剩下的部分为完全二叉树。
      • 性质： 假设二叉树深度是h，根结点在第一层。
        • (1)第k(1<=k<=h)层最多有2^(k-1)个结点
        • (2)整棵树有最多有(2^h)-1个结点
        • (3)在一棵二叉树中，叶子结点的数目比度为2的结点的数目多一个
        • (4)具有n个结点的完全二叉树的深度是h=floor(logn)+1=ceil(log(n+1)) //floor向下取整，ceil则相反
        • (5)对于一棵有n个结点的完全二叉树，按照从上往下 从左往右，从1开始编号。如果一个结点的编号是i 这个结点的 父亲i/2 左孩子2 * i 右孩子2 * i+1
      • 存储一棵完全二叉树时 只需要用顺序存储即可 用下标表示关系
  • 二叉树的存储方式
    • 1.顺序存储：完全基于一个普通的一维数组
      • 普通的二叉树可以补成完全二叉树，利用性质5进行顺序存储，用下标表示父子关系 缺点：浪费空间
    • 2.链式存储：直接用二叉链表存储
      • 操作：遍历 线索化

二叉树的遍历

  • 二叉树的遍历
    • 遍历：按照某种顺序访问数据结构中的所有的数据，每个数据仅访问一次。
    • 访问：对某个数据做某种操作—-输出 因为二叉树一个结点最多有两个后继结点，因此要确定遍历的顺序
  • 广度优先遍历：层次遍历
    • 假设根结点是第一层，从第一层开始，从左到右，再从上到下依次访问每个结点。
    • 遍历过程：基于队列
      • (1) 根结点入队
      • (2) 循环：只要队列非空，队首出队，访问队首，队首的孩子入队。
      • (3) 队列空，遍历结束
  • 深度优先遍历:
    • 把对一棵树的访问分成3部分：根结点 左子树 右子树
    • 先序遍历：根 左 右 基于递归: (1) 访问根结点 (2) 先序遍历左子树 (3) 先序遍历右子树
    • 中序遍历：左 根 右 基于递归: (1) 中序遍历左子树 (2) 访问根结点 (3) 中序遍历右子树
    • 后序遍历：左 右 根 基于递归: (1) 后序遍历左子树 (2) 后序遍历右子树 (3) 访问根结点
  • 题型: 先序+中序—->还原出二叉树
    • 由先序序列确定根结点 再到中序序列中确定左右子树
  • 后序+中序—->还原出二叉树
  • 层次+中序—->还原出二叉树
  • 若某二叉树的，层次abcdefgh 中序遍历顺序是dgbaechf，则后序遍历的访问顺序是什么。

二叉树的深度遍历(非递归版)

  • 先序遍历：根 左 右
    • 非递归：引入一个指针p和一个栈 辅助遍历。p初始指向根结点 只要p非空或者栈非空—–>说明还有结点没访问完
    • 循环：
      • if p非空， 说明当前要遍历以p为根的子树，先访问p，p入栈，p指向其左孩子p=p->l，继续循环
      • if p为NULL，说明栈顶结点的左子树访问完了，栈顶出栈 p指向栈顶的右孩子，继续循环
  • 中序遍历：左 根 右
    • 非递归：引入一个指针p和一个栈 辅助遍历。p初始指向根结点 只要p非空或者栈非空—–>说明还有结点没访问完
    • 循环：
      • if p非空， 说明当前要遍历以p为根的子树，先访问p的左子树，p入栈，p指向其左孩子p=p->l，继续循环
      • if p为NULL，说明栈顶结点的左子树访问完了，栈顶出栈 访问该结点，p指向栈顶的右孩子，继续循环
  • 后序遍历：左 右 根
    • 非递归：引入两个指针p和pre和一个栈 辅助遍历。p初始指向根结点。pre记录目前最后一个访问的结点 pre初始为NULL 只要p非空或者栈非空—–>说明还有结点没访问完
    • 循环：
      • if p非空，说明当前要遍历以p为根的子树，先访问p的左子树，p入栈，p指向其左孩子p=p->l，继续循环
      • if p为NULL，说明栈顶某棵子树访问完了， 通过pre来判断 如果栈顶有右子树并且pre不是栈顶右孩子 去访问其右子树。否则 只需要访问栈顶即可

非递归的代码和思路不唯一，但是一定要掌握一种，能独立写出来

二叉树的线索化

  • 中序遍历为例
    • 用比较简便快速的方法找到一个结点的前驱和后继
    • 利用每个结点中的空指针域，空的左指针域指向前驱，空的右指针域指向后继
  • 基于中序遍历来添加线索
    • 引入指针pre在遍历过程中一直保持上一个被访问的点。
    • 假设当前正访问的点是p
    • pre和p刚好是一对前驱和后继关系
    • 此时我们在访问p时 给p添加前驱线索 同时给pre添加后继线索

二叉排序树

  • 二叉排序树(二叉搜索树，二叉查找树：Binary Search Tree. BST)
    • 顺序性
  • 二叉树————》BST
    • 在二叉树的基础之上做如下规定：
      • 1.对于一个结点，如果他有左子树 左子树中的数据都比他小
      • 2..对于一个结点，如果他有右子树 右子树中的数据都比他大
      • 3.树中所有结点都满足以上规则。 性质：中序遍历序列有序
  • 查找 插入 删除
  • BST的存储结构：5种 链式存储最常用
    • 1.查找操作—–》最多大约查找logn（树的高度）
      • 在一棵BST中查找数据x
      • 如果树空树 不需要查找 直接返回空
      • (1）和根结点比较 如果相等找到，否则（2）/（3）
      • (2）如果x比根结点小 去根结点的左子树中查找
      • (3）如果x比根结点大 去根结点的右子树中查找
    • 2.插入操作 保证不插入重复数据
      • 在一棵BST中插入数据x
      • 如果是空树 创建结点s s->data=x 返回s即可 也就是让是做根结点即可 非空树：
      • 如果 x比根结点小 去根结点的左子树中插入x
      • 如果 x比根结点大 去根结点的右子树中插入x
      • 直到在合适的位置插入x位置
    • 3.删除操作—-保证没有重复数据
      • 在一棵BST中删除数据x—-》删除数据所在的结点
      • (1) 查找 找到x所在的结点p 及其父亲Pre
      • (2)根据p的度 分三类处理
      • p的度是0:把该结点直接删掉（free (p)；p=NULL，pre指向p度指针也要赋NULL）
      • p的度是1: 直接让pre指向p的指针 指向p唯一一个非空的孩子即可。free (p)；p=NULL
      • p的度是1或者0：都看成度为1，让唯一的孩子继承p的一切关系 声明一个孩子指针ch：如果p左孩子非空 ch指向其左孩子，否则指向右孩子 直接让pre指向p的指针 指向ch。free (p)；p=NULL
      • p的度是2: 让其前驱或后继结点t继承p的一切关系
        • 先找到t，p->data=t->data; 问题转化为在p的子树中删除t，并且t的度一定是0或者1，转化为情况1
  • 基于递归: BST——》进行优化 让其高度稳定在logn的级别

平衡二叉树(AVL)

  • BST的缺点:
    • 我们希望建立二叉排序树之后，使其进行查找操作的效率稳定在logn级别 但是如果用于建树的数据本身是有序的， 则建好的BST为一棵斜树，其高度为n 使得查找操作的效率退化
    • 对其进行优化，使得树的高度在任何情况下都能稳定在logn级别。 想办法让BST变得“矮胖”：即每个结点的左右子树的高度相差不大/左右子树中结点的个数相差不大，这样 整颗树的 高度就会变得更矮，提高查找效率
  • 顺序性
  • 平衡性：每个结点的平衡因子的绝对值<=1 二叉树——–>BST——–>平衡二叉树 (二叉平衡树 AVL树 )
    • 某结点 (p) 的平衡因子: 该结点的左子树 与 该结点的右子树的 高度差 1
    • 高度: 一棵树的高度其实就是其根结点的高度 结点的高度=max(左孩子的高度, 右孩子的高度)+1;
    • 平衡二叉树的存储方式: 5种 主要用链式存储 添加操作: 在一棵平衡二叉树上，插入数据x。
    • (1) 先执行BST的递归插入过程，把x插入到树上，满足顺序性。
    • (2) 考虑是否满足平衡性：被插入结点的爷爷及其以上的直系祖先，可能失衡。
      • 问题一: 怎么判断一个结点/一棵子树是否失衡: 平衡因子的绝对值和1比较
      • 问题二: 如果一棵子树失衡了，怎么调整？ 旋转
      • 最小失衡子树至少是3层: 先看最小的情况: 失衡结点x 插入结点s 刚好是爷孙关系
        • 1.插入结点s 在失衡结点x的左孩子的左子树上 (LL)
        • 要对失衡结点x 执行一次右旋。
        • y指向x的左孩子 y=x->l
        • y的右孩子变成x的左孩子
        • x->l=y->r;
        • x变成了y的右边孩子
        • y->r=x
        • 返回新的根结点
        • return y
      • 2.插入结点s 在失衡结点x的右孩子的右子树上 (RR)
        • 要对失衡结点x 执行一次左旋。
        • y指向x的右孩子
        • y=x->r
        • y的左孩子变成x的右孩子
        • x->r=y->l;
        • x变成y的左孩子
        • y->l=x
        • 返回新的根结点
        • return y
      • 3.插入结点s 在失衡结点x的左孩子的右子树上 (LR)
        • y指向x的左孩子
        • y=x->l
        • 对y调用左旋的函数
        • 再对x调用右旋的函数
      • 4.插入结点s 在失衡结点x的右孩子的左子树上 (RL)
        • y指向x的右孩子
        • y=x->r
        • 对y调用右旋的函数
        • 再对x调用左旋的函数
    • 删除操作: 在一棵平衡二叉树上，删除数据k。
      • （1）先执行二叉排序树的递归的删除过程 保证顺序性
      • （2）在递归返回的过程中 依次判断该层结点是否失衡，进行调整 失衡情况还是插入的4种情况：LL LR RR RL
      • 思路：左边删除一个结点 “相当于” 右边插入一个结点—-通过比较高度来做具体确定—-》RR/RL
      • 右边删除一个结点 “相当于” 左边插入一个结点—-通过比较高度来做具体确定—》LL/LR 如果高度相等 看成单旋的情况
    • AVL树的查找效率稳定在logn 缺点：不适用于动态查找
    • 树形逻辑结构: 普通的树 二叉树 二叉排序树 平衡二叉树
    • 算法：并查集 哈夫曼树

并查集(DSU/Union-Find)

  • 基于树形结构，处理不相交集合的合并和查询的问题。
  • 我们有一些正整数数据集合，对于这些集合存在两个操作：
    • 1:给出两个数据 x，y，把x所在的结合和y所在的集合合并
    • 2:给出两个数据 x，y，问俩数据是否在同一个集合中
    • 给出n个数据，初始时每个数据自成一个集合。
    • 给出若干操作序列:
      • op x y
      • 1 4 6
      • 1 7 2
      • 2 4 7
      • 2 1 4 6 7 3
  • 用树存储进行存储，实现合并和查询操作的集合—-》并查集
  • 两个集合的合并，就是一棵树的根结点做另一棵树的根结点的父亲。即一棵树成为另一棵树的子树
  • 查询：找到两个数据的根结点，如果根结点相同，则属于一个集合
    • 树：双亲
    • fa[y]=p; y的父亲是p
    • fa[x]=-1;-1代表不存在

哈夫曼编码(Huffman Tree)

  • 哈夫曼树 路径长度 带权路径长度

  • 最优二叉树:

    • 给定n个带权值的结点，要求这n个结点全部做叶子结点，可以额外再添加n-1个结点，去构造一棵只有度为2或者0的结点的二叉树，可以构造出多种形态。对每棵二叉树 都求一下WPL，其中WPL值最小的 称为最优二叉树/哈夫曼树

  • WPL= 求和 (叶子结点的权值*该结点到根结点的距离)

  • 为了使得WPL值变小，应该让权值大的点离根结点近。权值小的点 离根结点远。

  • 哈夫曼:

    • 一开始认为n个带权值的结点 各自是一棵树。每次选两棵树合并: 选根结点的权值最小的树合并。
    • 合并时: 添加一个新结点, 去做原来两棵树的根结点的父亲。新结点的权值为两个孩子的权值之和。

  • 信息编码/数据压缩:

    • 定长编码: 每个字符 使用的二进制编码长度都相同。
    • ASCII码: 8bit
    • Unicode编码: 16bit
      • ABCDE
      • DEAACAAAAAABA 12个字符——>12*3bit 36位
    • 优点 简单
    • 缺点 浪费空间

  • 变长编码: 根据字符的出现频率，调整编码长度。出现频率高的，编码长度短。

    • 优点: 节省空间
    • 缺点: 复杂
    • 注意: 要满足前缀属性，在此基础之上，是的总编码长度越短越好

  • 前缀属性:一个编码 不能是另一个编码的前缀。

    • A:0 B:00 C:01 D:10 E:11
    • 10 11 00010 0000000—–>DEAAAD

  • 树形结构：从根到每一个叶子的路径，刚好满足前缀属性

  • 通过哈夫曼树可以得出变长编码—-哈夫曼编码：

    • 假设有n个不同的字符，对它们进行变长编码：
      • 1.统计每个字符出现/使用的频率（次数）
      • 2.每个字符做叶子结点，其频率做叶子结点的权值
      • 3.再添加n-1个结点 构造哈夫曼树
      • 4.从根结点开始 得出到每个叶子结点的路径，往左标1 往右标0，即可得出每个字符的编码。

  • 存树的方式：双亲表示法结合孩子表示法。基于数组存树

图的概念（Graph)

  • 图状结构——非线性的逻辑结构

  • 图G: 是由集合V和集合E构成的, 记为G=(V,E); V是顶点的有限非空集合, E是顶点偶对的有限集合, 顶点偶对称之为边。边是由顶点表示的。|V| 点数目。 |E| 边的数目

  • 术语: 图的分类:

    • 1.根据是否有自边和重边分类 我们以后研究的图 都是简单图 简单图: 图中没有自边和重边 非简单图: 图中有自边或者重边
      • 有向边/弧: 表示边的顶点偶对是有序的, <a,b>和<b,a>是不同的, 即a到b b到a是不同的边
      • 无向边: 表示边的顶点偶对是无序的, (a,b) 即a到b b到a是同一条边
      • 一条无向边 相当于 两条方向相反的有向边
    • 2.根据边进行分类 有向图: 所有的边都是有向边 无向图: 所有的边都是无向边
    • 3.根据边是否带权值进行分类 带权图(网): 边都有权值 无权图: 边都没有权值
      • 无权图 相当于 边权都相等的带权图
      • 边权都相等的带权图 相当于 无权图
      • 无向带权图 无向无权图 有向带权图 有向无权图

  • 一些概念：

    • 1.完全图: 图中任意两个点之间都存在一条边。

      • 假设图中有n个点
      • 有向完全图: n* (n-1)条边
      • 无向完全图: n* (n-1)/2条边

    • 2.端点: 顶点

    • 邻接点: 一条边所连接的两个端点, 互为邻接点

    • 3.顶点的度

      • 无向图: 顶点的度就是该点所连的边的数目
      • 出边: 以该点为弧尾的边
      • 入边: 以该点为弧头的边
      • 有向图: 顶点的出度 出边的数目; 顶点的入度 入边的数目; 出度和入度之和 就是该点的度
      • 假设有一个图, 图中有n个点 e条边, 该图中n个点的度之和是2e

    • 4.子图 图G=(V,E); 图G1=(v1,e1);如果v1是V的子集, e1是e的子集并且e1中的边的端点不超过v1的范围。称G1是G的子图

    • 5.路径和路径长度

      • 路径: 是一个顶点到另一顶点所经过的顶点序列
      • 无权图的路径长度: 是一个顶点到另一顶点所经过的边的数目
      • 带权图的路径长度: 是一个顶点到另一顶点所经过的边权值之和
      • 简单路径: 路径的起点和终点可以相同, 其他点不能相同

    • 6.回路/环

      • 一条路径的起点和终点相同, 称为回路/环。其中如果只有起点和终点相同, 称为简单回路
      • 图可以根据是否存在环
      • 带环图 无环图

    • 7.稠密图: 边很多, 接近于完全图的图

      • 稀疏图: 边很少

    • 8.连通 连通图 连通分量: 无向图

      • 点的连通: 如果两个点可以相互到达, 称这两个点连通
      • 图: 如果图中任意两个点都连通, 称为连通图
      • 连通分量(极大连通子图): 从一个点出发 把能互相到达点的点和边都加进来, 构成一个极大连通子图
      • 强连通 强连通图 强连通分量: 有向图
      • 点的强连通: 如果两个点可以相互到达, 称这两个点强连通
      • 图: 如果图中任意两个点都强连通, 称为强连通图
      • 强连通分量(极大强连通子图): 从一个点出发 把能互相到达点的点和边都加进来, 构成一个极大强连通子图

    • 9.生成树: 是图的极小连通子图: 用尽可能少的边, 使得全部的点连通。

      • 连通图才有生成树

      图的存储方式

  • 图状结构——非线性的逻辑结构

    • 图G: 是由集合V和集合E构成的, 记为G=(V,E); V是顶点的有限非空集合, E是顶点偶对的有限集合, 顶点偶对称之为边。边是由顶点表示的。|V| 点数目。 |E| 边的数目

  • 图的存储结构:

    • 有向图/无向图: 边集数组 邻接矩阵 邻接表
    • 有向图: 十字链表
    • 无向图: 多重邻接表

  • 图的存储：主要就是存顶点数据和边 假设有n个顶点，m条边

    • 顶点数据：数组存顶点
    • 1.边集数组 顶点数据：数组存顶点
      • 边：边是由顶点表示的，存边的两个端点即可
      • 一条边用一个结构体表示。
      • 无向边 struct Node { char x,y; int w; }g[];
      • 有向边 struct Node { char satrt,end; int w; }g[];
    • 2.邻接矩阵：适用于稠密图 稀疏图会比较浪费空间
      • 顶点数据：顶点数组存顶点
      • v[i]=’A’. 顶点数组中的下标就是点的编号 边：二维数组
      • 无权有向图：二维数组来存边
      • g[i] [j]=0. 没有i—>j的边
      • g[i] [j]=1 有i—>j的边
      • 带权有向图：边权不超过10000。inf=10001
      • g[i] [j]=w; 有i—>j的,边权为w的边
      • g[i] [j]=inf(∞) 没有i—>j的边
      • 无权无向图：二维数组来存边
      • 要把一条无向边当成两条方向相反的有向边存
      • 不存在一条i j之间的无向边
      • g[i] [j]=g[j] [i]=0;
      • 存在一条i j之间的无向边
      • g[i] [j]=g[j] [i]=1;
      • 带权无向图: 要把一条无向边当成两条方向相反的 权值相等的有向边存
      • 不存在一条i j之间的无向边
      • g[i] [j]=g[j] [i]=inf;
      • 存在一条i j之间边权为w的无向边
      • g[i] [j]=g[j] [i]=w;
    • 3.邻接表：稀疏图用的比较多 稠密图也用
      • 顶点数据：顶点数组存顶点
      • v[i]=’A’. 顶点数组中的下标就是点的编号
      • 边:
      • 无向图：把每个点邻接点 构成一个链表 挂在该点之后。
      • 有向图:
      • 邻接表：把每个点的出边邻接点 构成一个链表 挂在该点之后。
      • 逆邻接表：把每个点的入边邻接点 构成一个链表 挂在该点之后。
    • 4.十字链表：对有向图的邻接表和逆邻接表合二为一的一种优化
      • 对于有向图，邻接表和逆邻接表 并非全能
      • 为了即方便找出度 又方便找入度，可以把邻接表和逆邻接表 合二为一，一个顶点后面挂两个链表 还是有缺点：（1）浪费空间，一条边对应两个链表结点。（2）操作麻烦
      • 进行优化：一条边只对应一个结点 把同一个结点插入到两个不同的链表中
      • 十字链表的结点结构 | tail(弧尾) | tnext(指向弧尾下一条出边的指针) | head(弧头) | hnext(指向弧头下一条入边的指针) |
    • 5.多重邻接表：无向图的邻接表，本身就是一条边在两个链表中对应两个结点，存两次 有缺点：（1）浪费空间，一条边对应两个链表结点。（2）操作麻烦
      • 仿照有向图的十字链表，进行优化：一条边只对应一个结点 把同一个结点插入到两个不同的链表中 得到 无向图的多重邻接表
      • 多重邻接表的结点结构： | xi(端点x) | xnext(指向x的另一条边) | yi(端点y) | ynext(指向y的另一条边) |

图的广度遍历(BFS)

  • 图的遍历: 深度优先遍历: 广度优先遍历 (Breadth First Search) BFS:
  • 可以根据邻接关系, 人为的划分层次, 起点上第0层, 第0层的邻接的是第1层。。。 第i层的邻接的是第i+1层。 广度优先遍历就类似树的层次遍历
  • 遍历思路:
    • (1) 任选一个未被访问的点做起点, 起点入队并且打标记
    • (2) 循环操作: 只要队列非空, 队首q出队, 访问q, 把q的未入队过的邻接的入队并且打标记
    • (3) 直到队列空为止。如果是连通图就结束, 如果是非连通图, 继续执行(1) 直到所有的点被访问一次为止。
  • 1.存图方式: 邻接矩阵/邻接表
    • 以邻接表为例 对无向图进行BFS
  • 2.如果是非连通图应该怎么办?
    • 只需要在BFS外套一层循环即可, 也就是遇到没被访问过的点 就以其为起点执行BFS
  • 3.入队: 顶点数据入队 还是把 顶点的下标入队: 下标入队
  • BFS可以求无权图的最短路径
