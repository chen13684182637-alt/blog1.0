数据结构概述

  • 所谓数据结构就是把现实世界里的东西抽象到计算机里，通过计算机中数据集合与关系集合抽象出逻辑结构最后做成存储结构，那么现实世界里的具体不同问题就需要通过不同的算法与数据结构共同构成一个程序来解决

顺序表及其代码实现

  • 线性逻辑结构

    • 一般的线性表：是具有相同的数据类型的n(n>=0)个数据元素的有限序列，n称为表长，当n=0时线性表是空表。
    • L=(a1,a2,a3,a4…an)

  • 操作：增 删 改 查

  • 线性表的存储/物理实现：

    • 顺序存储：基于顺序存储结构实现的线性表 称为顺序表： 用数组来存储线性表
    • int data[maxx];maxx>=n
    • int size;//记录线性表（顺序表）的实际的元素个数

  • 初始化一个空的顺序表：

    • 申请数组所占空间，给size赋初始值

  • 增：

    • 1.在末尾顺序添加：把新数据放在size位置，size++
    • 2.在指定位置i(0<=i<=size)添加一个数据
      • 1）把原来的i~~size-1往后移动一个位置 空出i位置
      • 2）data[i]=k;

  • 查找数据k所在的位置

  • 删除数据k：

    • 1.查找k
    • 2.删除k

  • 优缺点：

    • 优：1.随机访问 首地址+偏移量直接访问某个数据 data[i] 2.存储密度高

    • 缺点：操作繁琐 浪费时间

单链表及其代码实现

  • 链表的结点结构：

    • 数据域：存放数据
    • 指针域：存放下一个数据所在的结点的地址
    • 首元节点：第一个数据所在的结点
    • 头指针：保存链表中第一个节点的地址
    • 头节点：保存首元结点地址的结点

  • 带头节点的单链表：

    • 使得对首元结点的操作和对其他结点的操作 统一起来
    • 使得空链表和非空链表统一起来

  • 不带头节点的单链表

  • 操作：增 删 改 查

    • 1、初始化：创建一个带头结点的空链表L。
      • 声明一个头指针L，创建一个节点空间 指针域赋NULL，头指针L指向该结点空间。
    • 2、增：在链表中添加一个数据k：先新建一个结点s，把数据k放到结点s的数据域中，然后把结点s插入到链表中
      • 头插法：在首元结点前面插入结点s。（1）s的指针域指向原首元结点。（2）头结点的指针域指向新结点s。
      • 尾插法：在尾结点后面插入结点s。（1）先找到尾结点。（2）s的指针域等于尾结点的指针域。（3）尾结点指向s。
      • 在指定位置插入：在数据x之后插入k。（1）找到x所在结点p。（2）s的指针域等于结点p的指针域（3）结点p指向s
    • 3、查找：查找数据x所在的结点，能找到则返回结点的地址，否则返回NULL；
    • 4、删除：在链表中删除数据k
      • 找到k所在的结点p及其上一个结点q
      • q的指针域指向p的下一个结点
      • 释放p结点

  • 优：灵活 插入删除比顺序表快/适用于动态场景

  • 缺：不能随机访问

双向链表和循环列表及其实现

  • 双向链表：在单链表的的基础之上，给每个结点增加一个指针域，用于保存上（前）一个结点的地址
  • 双向链表的结点结构：pre data next
  • 在有头结点的双向链表L中：
    • 查找：查找数据x所在的结点 和单链表查找一摸一样
    • 修改：和单链表修改一摸一样
    • 增：
    • 删：
  • 因为每个结点都多了pre多几步对pre处理，注意1.不要断链2.注意涉及到的结点是否一定存在
  • 循环链表：循环单链表 循环双链表
  • 带头结点的循环单链表：带头结点的单链表 首尾相接 最后一个结点的next指针域指向头结点
  • 带头结点的循环双链表：带头结点的双链表 首尾相接 最后一个结点的next指针域指向头结点 头结点pre指针域指向最后一个结点
  • 线性表：顺序表/链表

注意：在实现的时候着重把单双链表的思路理解并记忆，能独立复现，剩下的循环自然也就理解，每个结构实现的时候不要去复制粘贴之前写过的代码，每次都从头开写。

栈及其代码实现(Stack)

  • 栈（Stack）—线性逻辑结构

  • 1.线性表：一对一的线性逻辑结构，可以在任意位置执行任何操作

  • 2.栈：是只允许在一端进行删除或者插入操作的线性表。并且没有查找和修改操作。

    • 术语/概念：
    • 栈顶（Top）：允许执行删除or插入操作的一端
    • 栈底（Bottom）：不允许执行删除or插入操作的一端
    • 空栈：不含任何数据。

  • 性质：先进后出/后进先出

  • 应用场景：

    • 1.函数的调用的实现（递归）
    • 2.业务需求：一些倒着走的逻辑：历史记录，撤销…
    • 3.物理上的堆栈

  • 操作：

    • 初始化一个空栈
    • 入栈：往栈顶放一个数据
    • 出栈：删除栈顶数据

  • 判满

  • 判空

  • 得到栈顶元素（get）

  • 栈的存储结构的实现：

  • 基于顺序存储结构实现：顺序栈——基于数组实现a[maxx]

    • 根据栈顶指针top含义不同 有两种实现方法：

      • 1.栈顶指针top标记真正的栈顶数据：

        • 初始化：声明一个数组，同时把栈顶“指针”top（本质上是下标）赋值为-1.
        • 入栈：先判满，把数据k入栈top++，a[top]=k;
        • 出栈：先判空，top– –；
        • 判满：栈有没有充满整个数组
        • 判空：top= =-1
        • get栈顶元素：先判空，非空时return a[top]

      • 2.栈顶指针top标记真正栈顶数据的下一个位置，即下一个入栈数据的下标：top-1才是真正的栈顶数据

        • 初始化：声明一个数组，同时把栈顶 “指针”top（本质上是下标）赋值为0。

        • 入栈：先判满，把数据k入栈：a[top] = k; top++;

        • 出栈：先判空：top--;

        • 判满：判断栈是否充满整个数组：top == maxx

        • 判空：top == 0

        • get（获取栈顶元素）：先判空，非空时return a[top-1]

  • 基于链式存储结构(带头结点的单链表)实现：链式栈
    • 栈顶指针：头指针，非空时，首元结点是栈顶。尾结点是栈底
    • 初始化：声明一个带头结点的空链表
    • 入栈：头插法
    • 出栈：先判空，删除首元结点
    • 判空：top->next == NULL
    • 上溢：顺序会有，链式不会有
    • 下溢：顺序和链式都会有
  • 一道考研题：
  • C++STL库里面的stack和这里的栈接口和行为逻辑是一样的。

队列及其代码实现（Queue）

  • 队列：只允许在一端进行插入操作，另一端进行删除操作的线性表。并且没有查找和修改操作。

  • 队尾：执行插入操作的一端

  • 队首：执行删除操作的一端

  • 空队列：不含任何数据元素

  • 性质：先进先出 / 后进后出

  • 队首 “指针”

  • 队尾 “指针”

  • 入队 出队 判满 判空 初始化

    1. 队列的顺序存储：顺序队列 — 数组 maxx 7

  • data[maxx];

  • 队首指针 int f = 0

  • 队尾指针 int r

  • 单向队列

    • (1) 队尾指针 r 指向真正的队尾元素

      • 初始化: f=0 r=-1
      • 入队: r++ ,data[r]=k;//data[++r]=k
      • 出队：删除队首指针指向的数据。f++
      • 判满：判断数组还能否容纳数据。 r==maxx-1 (可能存在假满) . 假溢出 —– 循环队列
      • 判空: f==r+1

    • (2) 队尾指针 r 指向真正的队尾元素的下一个位置，r-1 是真正的队尾

      • 初始化: f=0 r=0
      • 入队: data[r]=k ,r++. //data[r++]=k
      • 出队：删除队首指针指向的数据。f++
      • 判满：判断数组还能否容纳数据。 r==maxx (可能存在假满) . 假溢出 —– 循环队列
      • 判空: f==r

  • 循环队列：基于循环数组的：通过数学上的取模运算 实现下标的首尾相接

    • 队尾指针 r 指向真正的队尾元素的下一个位置，

    • 初始化: f=0 r=0

    • 入队: data[r]=k,r=(r+1)%maxx,

    • 出队：删除队首指针指向的数据。f=(f+1)%maxx,

    • 判满：

      • (1) 引入变量 sum 记录队列中数据个数

        • r==f && sum==maxx 满

        • r==f && sum==0 空

      • (2) 引入一个标记变量flag==0 出队 flag==1 入队

        • r==f&&flag==1 满。

        • r==f&&flag==0 空

      • (3) 牺牲一个空间，数组中有maxx-1个数据时就满了

        • f== (r+1) %maxx 满

        • r==f 空

  • 2.队列的链式存储：链式队列—–基于带头结点的单链表

    • 首元结点一侧 应该是队首
    • 尾结点一侧 应该是队尾
    • 用头指针做队首指针即可，此时队首指针指向真正队首的上一个结点
    • 队尾指针：指向尾结点的尾指针即可
    • 初始化：搞一个头结点 让队首指针和队尾指针都指向头结点即可
    • 入队：尾插法
    • 出队：删除首元结点 判空：f==r // f->next==NULL

  • 队列的应用场景：一些需要排队的需求

双端队列及其代码实现

  • 特殊的队列：两端均允许入队操作和出队操作，灵活
  • 不分队首和队尾
  • 通过四种操作的搭配 可以搞一些特殊的结构
    • 例如：左边入队的数据必须从左边出队。右边入队的数据必须从右边出队

  • 初始化 左端入队 左端出队 右端入队 右端出队 判满 判空

  • 基于顺序存储结构实现双端队列—循环数组data[maxx] 左端“指针” l 右端“指针” r

    • 初始化：l=0 r=0 sum=0

    • 左端入队：数据k左端入队sum++

      • 如果认为 l指向真正的左端数据l=(l-1+maxx)%maxx.data[l]=k;————————1
      • 如果认为 l指向真正的左端数据前一个位置data[l]=k,l=(l-1+maxx)%maxx———-2

    • 右端入队：数据k右端入队sum++

      • 如果认为 r指向真正的右端数据：r=(r+1)%maxx, data[r]=k;—————————3
      • 如果认为 r指向真正的右端数据后一个位置：data[r]=k, r=(r+1)%maxx————-4

    • 只能选14和23，原因画图可知

    • 左端出队：l=(l+1)%maxx sum–;

    • 右端出队：r=(r-1+maxx)%maxx sum–;

    • 判满判空：引入一个变量sum记录队列中数据的个数

      • if(sum==0)空
      • if(sum==maxx)满
  • 基于链式存储结构实现双端队列—–双向链表
    • 初始化: 创建一个带头结点的空的双向链表, l r都指向头结点
    • 认为 l指向真正的左端数据, 如果认为 r指向真正的右端数据后一个位置
    • 左端入队: 新建一个结点s , s->data=k,s插入到l的前边,
    • 右端入队: r->data=k, 新建一个结点s s插入到r后边, 此时s中是脏数据
    • 左端出队: 把l指向的结点free
    • 右端出队: 把r指向的结点free 判空: l==r

线性逻辑结构完结

  • 线性逻辑结构:
  • 线性表: 顺序表 链表 (单 双 循环)
  • 栈: 顺序栈。链式栈
  • 队列: 顺序队列 (单向顺序—>循环队列) 链式队列
  • 双端队列: 顺序存储 (循环数组) 链式存储 (双向链表)
